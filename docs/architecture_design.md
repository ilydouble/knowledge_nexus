# AI 语义网盘（Knowledge Nexus）架构设计文档

## 1. 设计哲学

本系统不再将文件视为孤立的二进制对象，而是将其视为知识节点。系统通过三层解耦的架构，平衡组织的“秩序与管控”与个人的“灵感与偏恋”。

核心逻辑是：物理存储受控，逻辑链接自由。

## 2. 核心架构：三层本体模型

我们将知识结构拆解为三个并行存在的逻辑层。

### 2.1 顶层知识层（L1: Standard Ontology）：“秩序”

定义全局公认的概念、实体类型及相互关系，例如什么是“项目”、什么是“交付物”、什么是“技术路线”。

管理方式：

- 由系统管理员或领域专家维护。
- 具有强一致性。
- 通过审批、版本和变更记录保障全局稳定。

核心功能：

- 提供标准元数据模板。
- 提供受控词表（Taxonomy）。
- 确保全员在同一个语境下工作。

### 2.2 分类知识层（L2: Collective Domain）：“共识”

定义部门、团队或项目组共有的知识资产，例如研发部的“红外技术知识库”、市场部的“竞品分析库”。

管理方式：

- 由具有权限的成员共同编辑。
- 支持版本控制、评审和回滚。
- 与组织空间、项目空间和团队权限绑定。

核心功能：

- 实现知识传播与复用。
- 通过共享图谱展示项目全貌。
- 承载团队共识、项目脉络和跨文件关系。

### 2.3 个人偏好层（L3: Personal Cognition）：“偏恋”

定义用户个人的主观联系、私人笔记和跳跃式思维，例如“我觉得项目 A 的逻辑可以参考科幻电影 B”。

管理方式：

- 绝对私有。
- 存储在用户的个人语义索引空间。
- 默认不参与团队共享，除非用户主动发布或转化为 L2 知识。

核心功能：

- 提供仿 Obsidian 的双向链接核心。
- 用户可以在不改变文件物理位置的情况下，建立跨部门、跨领域的私有逻辑连线。
- 支持 `[[Link]]`、私人标签、局部图谱、卡片化笔记和个人 RAG 记忆。

## 3. 技术架构组件

### 3.1 存储与处理引擎

对象存储（Object Storage）：

- 存储物理文件。
- 兼容 S3 协议。
- 负责文件级权限、租户隔离、版本、归档和生命周期策略。

图数据库（Graph Database）：

- 存储节点间的关系。
- L1/L2 关系存储在全局图或团队图中。
- L3 关系存储在用户私有图分片中。
- 可选型包括 Neo4j、NebulaGraph、JanusGraph。

向量数据库（Vector DB）：

- 存储文件内容、摘要、切片、实体和卡片的语义向量。
- 支持 RAG、相似文件推荐、语义搜索和自动链接。
- 可选型包括 Milvus、Pinecone、Weaviate、Qdrant。

### 3.2 AI 语义提取模块

Multi-modal Parser：

- 自动解析 PDF、Office 文档、图像、音频和视频。
- 提取文本、结构、表格、图片描述、实体和事件。

GraphRAG Engine：

- 结合图谱的检索增强生成。
- 用户提问时，AI 不仅检索文本片段，还检索其在图谱中的邻居节点。
- 支持按 L1/L2/L3 视图过滤上下文。

Auto-Linker：

- 根据顶层知识结构、团队知识图谱和个人偏好图谱，自动为上传文件推荐关联。
- 推荐类型包括实体对齐、相似文件、项目关联、前置/后置依赖和个人双向链接。

## 4. 关键业务流程

### 4.1 知识录入与自动重构

1. 上传：用户将文件上传至物理文件夹，系统完成权限初审。
2. 入库：对象存储保存原始文件，元数据服务登记文件实体。
3. 解析：AI 自动识别文件中的实体，包括人、事、地、物、组织、项目和技术概念。
4. 对齐：AI 将识别出的实体与 L1 顶层知识对齐。
5. 建图：系统将标准关系写入 L1/L2 图谱，将个人关系写入 L3 私有图谱。
6. 建索引：系统生成文本索引、向量索引和图索引。
7. 建议：系统询问用户：“此文件与您上周收藏的个人偏好 A 逻辑相似，是否建立链接？”

### 4.2 跨层级检索

当用户搜索“红外传感器”时：

- 顶层视图：返回标准的技术定义、实体类型和分类路径。
- 分类视图：返回公司目前正在进行的红外项目文件、团队沉淀和共享图谱。
- 个人视图：返回用户自己的标注、双向链接、私人卡片和主观联想。

检索结果必须经过权限过滤。用户无权打开的文件可以在图谱中显示为“加密节点”，但不能泄漏文件内容、标题敏感字段或预览片段。

## 5. 权限与安全模型

为了满足“老板要权限，我要自由”的需求，系统采用视图过滤机制。

| 维度 | 文件夹权限（Physical） | 知识图谱权限（Logical） |
| :--- | :--- | :--- |
| 管控对象 | 原始文件存储、下载、预览 | 关系链、标签、注释、链接的可见性 |
| 可见策略 | 严格按照组织架构与空间权限 | 节点可见性、边可见性和属性可见性分层控制 |
| 冲突处理 | 若无权限查看文件 A，则无法打开 | 图谱中可显示 A 为加密节点，但只展示允许暴露的关系轮廓 |

安全原则：

- 文件权限永远优先于知识图谱权限。
- 图谱关系不能反向泄漏文件内容。
- L3 私有数据默认仅本人可见。
- L3 转 L2 必须经过用户确认和团队空间权限校验。
- AI 检索上下文必须先过滤权限，再进入模型。

## 6. 用户交互界面设计

### 6.1 双重侧边栏

左侧是传统文件夹树，用于管理物理资产、空间、团队目录和权限边界。

右侧是知识检查器，用于展示当前文件的 L1/L2/L3 关联关系、实体、摘要、双向链接和推荐连接。

### 6.2 力导向图

图谱视图采用三色节点：

- 蓝色：官方分类，对应 L1 顶层知识。
- 绿色：共享知识，对应 L2 团队或项目共识。
- 紫色：个人私有，对应 L3 个人认知。

深度滑块用于控制图谱叠加程度：

- 只看官方结构。
- 叠加团队共识。
- 叠加个人偏好。
- 展开指定深度的邻居节点。

### 6.3 原子化卡片

系统模仿 Obsidian 的 `[[Link]]` 语法。用户在任何编辑区域输入 `[[` 即可引用网盘内文件、知识节点、实体、项目、卡片或笔记。

## 7. 演进路线图

### Phase 1：MVP

- 基础物理网盘。
- 文件上传、下载、预览和权限隔离。
- AI 自动标签。
- 个人双向链接。
- 基础语义搜索。

### Phase 2：Graph Integration

- 引入图数据库。
- 实现全局图谱可视化。
- 建立 L1 顶层知识定义。
- 支持 L2 团队知识库和项目知识图谱。

### Phase 3：AI Agent

- 引入 GraphRAG。
- AI 能够基于三层知识结构提供深度决策建议。
- 支持自动知识巡检、冲突发现、知识缺口提醒和跨项目经验复用。

## 8. 设计者批注

该架构的核心突破点在于语义叠加层。它允许组织保留传统的行政管理边界，也就是文件夹权限；同时允许知识在语义层面实现跨越边界的自由流动，也就是图谱链接。

这不仅是一个网盘，更是一个随着组织和个人共同成长的动态数字大脑。
