Knowledge Nexus Architecture

整体架构设计与当前实现阶段

Knowledge Nexus 是一个以 Cloudreve 为物理文件底座、以语义图谱和向量检索为知识叠加层的 AI 语义网盘。系统目标是让组织继续严格管理文件，同时允许知识关系在权限边界之上自由生长。

1. 架构核心命题

本项目的核心原则是：物理存储受控，逻辑链接自由。原始文件、下载、预览和硬权限仍由 Cloudreve、对象存储与组织空间管理；摘要、标签、实体、双向链接、图谱关系和 GraphRAG 上下文由 Nexus 作为旁路语义层负责。

Cloudreve / 文件系统
文件、目录、权限、事件
→
Nexus API / Worker
录入、解析、链接、检索
→
知识存储层
Postgres、Neo4j、Milvus

2. 分层设计

L1 顶层知识层

承载标准本体、实体类型、受控词表和全组织一致的知识定义。它是系统里的“秩序层”，未来应由管理员或领域专家维护。

Standard Ontology 全组织可见

L2 团队知识层

承载团队、项目、部门形成的共识图谱。它连接文件、项目、交付物、人员和经验沉淀，是协作知识库的主体。

Collective Domain 团队可见

L3 个人认知层

承载个人笔记、私人标签、主观联想和 [[Link]] 式双向链接。默认仅本人可见，主动发布后才进入团队知识层。

Personal Cognition 个人私有

3. 主要组件职责

组件 当前职责 代码位置
Web Console 提供 MVP 演示界面，支持输入 Cloudreve URI、索引文本、查看知识检查器和 GraphRAG 结果。 apps/web/src/App.jsx
FastAPI Gateway 暴露健康检查、Cloudreve 绑定、录入同步、演示索引、链接创建、图谱邻域、语义搜索和 GraphRAG 问答接口。 nexus/app_factory.py
Repository Port 以协议形式抽象任务、文档、链接、图谱数据的读写，当前有内存实现和 Postgres 实现。 nexus/repositories/
Semantic Processor 实现轻量文本规范化、分块、摘要、关键词标签和大写实体抽取，用于证明语义索引闭环。 nexus/services/semantic.py
Link Service 创建 L1/L2/L3 知识链接，并根据层级推导默认可见性和 owner scope。 nexus/services/links.py
Auto-Linker 基于共享标签生成 L3 自动链接候选，作为后续向量相似度和图谱推理的占位实现。 nexus/services/autolinker.py
Permission Filter 执行“先授权、后检索、再生成”的最小权限模型；不可访问节点会被替换为加密节点。 nexus/services/permissions.py
GraphRAG Service 按知识层过滤图谱结果，并只使用权限过滤后的可见节点生成演示答案和引用。 nexus/services/graphrag.py
Cloudreve Worker 监听 Cloudreve SSE 文件事件，将可解析事件转为 ingestion job。 nexus/worker.py

4. 数据与调用流程

  1. 用户或 Cloudreve 事件提交文件 URI，Nexus 创建 IngestionJob。
  2. 演示索引接口接收文本内容，TextParser 生成摘要、标签、实体和文本分块。
  3. Repository 将语义文档、分块、知识链接持久化到内存或 Postgres。
  4. 图谱视图由文档节点和 KnowledgeLink 派生，Neo4j adapter 已提供文件节点与关系的 upsert 和一跳邻域查询能力。
  5. Milvus adapter 已提供 chunk 向量写入与相似搜索接口，等待与真实 embedding 流水线更深集成。
  6. 检索和 GraphRAG 先按 L1/L2/L3 层级裁剪，再经过权限过滤，最后返回给 Web Console 或调用方。

6. 当前部署视图

MVP 推荐以 sidecar 方式运行：不重复搭建现有 kg-* 基础设施，而是让 Nexus API、Worker 和 Web 接入既有 Docker network。

Host 开发模式

  • Python 3.11，运行 uvicorn apps.api.main:app --reload。
  • Web 端在 apps/web 中通过 npm run dev 启动。
  • 默认 API 地址为 http://localhost:8000，Web 地址为 http://localhost:5173。

Docker Sidecar 模式

  • nexus-api 提供 API Gateway。
  • nexus-worker 监听 Cloudreve 文件事件。
  • nexus-web 提供前端控制台。
  • 外部依赖通过 kg-network 连接到 Postgres、Redis、Neo4j、Milvus 和 MinIO。

7. 下一阶段建议

  1. 补齐 Cloudreve 文件内容读取、用户身份映射、空间权限同步和事件幂等处理。
  2. 把语义处理从 demo-index 推进到真实 ingestion pipeline，支持文件解析、异步任务状态和失败重试。
  3. 将 Neo4j 与 Milvus 纳入主查询路径，形成关键词、向量、图邻域的混合检索。
  4. 接入真实 embedding 与大模型生成，完善 GraphRAG prompt、引用、权限边界和答案审计。
  5. 扩展 Web Console：文件树、知识检查器、图谱层级开关、自动链接确认和个人笔记入口。
  6. 把 L1 本体、L2 团队图谱、L3 私有图谱的治理流程落到数据模型和 API 上。
